# 网文小说创作 Agent 智能体 - 规格说明

## Why
面向网文作者（新手/兼职/专职），全链路辅助网文生产：选题→大纲→人设→章节正文→剧情润色→节奏把控→热点适配→合规自查。解决卡文、剧情逻辑崩、人设 OOC、不会写钩子爽点、长篇大纲难梳理、批量更新效率低、不懂平台规则等核心痛点。

## What Changes
- 新增前端项目（React + TypeScript + Tailwind CSS）
- 新增后端项目（Java 21 + Spring Boot）
- 实现 **1 总控 + 6 子 Agent** 多智能体协作架构
- 实现双存储层（结构化数据库 + 向量知识库）
- 实现网文专属能力（赛道适配、爽点设计、钩子制造、毒点规避、合规检测）
- 实现三种核心工作流（新建小说/续写章节/卡文修复）

## Impact
- 全新项目，无既有代码影响
- 需要 LLM API 集成（支持多模型切换）
- 需要向量数据库集成（Chroma/FAISS/Milvus）
- 需要外部工具插件扩展（违禁词检测、榜单爬虫等）

## 整体架构设计（多智能体分工协作）

### 架构概览
```
用户交互层
    ↓
总控调度 Agent（大脑，任务分发 + 上下文统筹）
├─ 1. 世界观&大纲规划 Agent
├─ 2. 人物塑造 Agent
├─ 3. 剧情爽点节奏 Agent
├─ 4. 正文写作分镜 Agent
├─ 5. 润色&文风校准 Agent
├─ 6. 合规&网文运营 Agent
    ↓
持久存储模块（向量知识库 + 结构化数据库）
    ↓
外部工具插件集
```

### 模块详细说明

#### 总控调度 Agent：中枢核心
**核心职能**：
- 解析用户自然语言需求，拆分细分任务分发给对应子 Agent
- 统一管理全局记忆，保证所有子 Agent 共享世界观、人设、已写剧情
- 冲突校验：检测新生成内容是否人设崩塌、伏笔矛盾、战力崩坏
- 多轮对话状态维护：记录当前创作阶段（选题/写大纲/写第一章/卡文修改）
- 汇总各子 Agent 输出，整合交付给用户

**内置提示词约束**：
- 强制优先读取向量库历史设定，所有生成内容必须和已有设定对齐
- 区分用户指令优先级：人设 > 世界观 > 主线剧情 > 临时剧情
- 识别用户诉求类型：新建小说/续写章节/改剧情/卡文求助/优化爽点

#### 六大垂直专业子 Agent

**1. 世界观&大纲规划 Agent**
- 赛道选题生成（玄幻/甜宠/无限/都市战神）
- 世界观搭建（力量体系、地域地图、势力划分、历史背景）
- 分级大纲（全书总纲→分卷大纲→每卷核心冲突→百章级长线伏笔）
- 剧情分支推演（多条剧情走向，标注爽度、受众适配度）
- 节奏框架模板（黄金结构：开篇金手指→第一次冲突→升级→大反派→阶段性高潮）

**2. 人物塑造 Agent**
- 存储结构化人物档案（基础/内核/能力/关系网/标志性台词）
- 批量生成主角/配角/反派/工具人
- 校验正文对话是否符合人设，自动修正崩人设台词
- 角色成长线设计（每一卷人物心态变化）
- 反派逻辑合理化，避免"为坏而坏"

**3. 剧情爽点节奏 Agent（网文核心差异化）**
- 内置网文底层规则库：
  - 黄金三章规则（前三章必须有钩子、冲突、金手指、期待感）
  - 章节结构模板（每章结尾留钩子，每 3-5 章一个小爽点，20 章一个小高潮）
  - 爽点分类库（打脸、逆袭、暴富、修为暴涨、美人倾心、反派后悔等）
  - 毒点规避库（圣母主角、降智女主、憋屈无反击、战力崩坏等）
  - 情绪曲线控制（压抑铺垫→爆发爽感→短暂留白→新冲突引入）
- 针对现有章节打分（爽度、钩子强度、节奏缺陷）
- 卡文救场（给出 3 种化解当前僵局的冲突方案）
- 高潮段落强化、调整剧情憋屈程度

**4. 正文写作分镜 Agent**
- 根据大纲、人设、节奏要求，生成完整章节正文
- 网文分段逻辑（短句、少大段文字，对话单独分行，适配手机阅读）
- 自动穿插细节（环境描写、心理活动、动作神态）
- 区分视角（第三人称全能视角/主角第一视角）
- 批量续写（支持一次性生成 3/5/10 章内容）
- 禁止过度文绉绉书面化语言，严格匹配赛道文风

**5. 润色&文风校准 Agent**
- 多档位改写（精简缩句、扩写填充细节、强化情绪、淡化狗血）
- 文风统一（前后章节用词、对话风格对齐）
- 消除重复描写（重复动作、重复心理、重复场景）
- 标题/简介/章节名生成（适配平台引流，制造悬念）
- 段落打磨（优化结尾钩子，提升追读欲望）

**6. 合规&网文运营 Agent**
- 合规检测能力：
  - 违禁词筛查（暴力、色情、涉政、敏感历史、不当价值观）
  - 网文红线校验（禁止过度血腥、校园霸凌美化、乱伦、抹黑公职等）
  - 自动替换敏感表述，给出修改方案
- 运营辅助能力：
  - 对标同赛道爆款，给出剧情优化建议
  - 读者评论模拟推演（预判读者吐槽点、喜爱点）
  - 推荐引流文案、书评、新书简介优化

## 持久存储层（长篇不崩设定的核心壁垒）

### 1. 结构化关系数据库（PostgreSQL）
**核心数据表**：
- `novel_info`：小说基础信息（赛道、书名、总卷数、核心金手指）
- `character_table`：全角色档案（基础/内核/能力/关系网/标志性台词/成长曲线）
- `volume_outline`：分卷、分章大纲、节点冲突
- `plot_timeline`：时间线、重大事件、战力等级体系
- `foreshadow_lib`：伏笔清单（预埋章节、回收章节、对应线索）
- `chapter_content`：已生成章节原文、修改记录

### 2. 向量知识库（Chroma/FAISS/Milvus）
**存储非结构化文本，支持语义检索**：
- 全文段落向量：续写时检索前文相似场景、人物互动，保持文风统一
- 赛道范文向量库：海量高分网文片段，作为风格参考
- 爽点/钩子案例库：语义匹配自动复用成熟剧情模板
- 用户历史修改偏好：记住作者喜欢的叙事风格、爽点类型

**读取逻辑**：
每次任意子 Agent 生成内容前，总控 Agent 先执行向量检索：
1. 检索当前出场人物完整人设
2. 检索前文相关剧情、已埋伏笔
3. 检索同赛道同类场景优质范文参考
4. 校验是否存在设定冲突，冲突则阻断生成，提示用户修正设定

- 违禁词检测 API：实时文本风控
- 网文榜单爬虫插件：自动抓取起点/番茄/七猫爆款标签、热门题材

## 完整创作工作流

### 流程 1：新建一本小说（从零创作）
1. 用户输入需求：赛道 + 偏好（例：男频都市，重生商界，低调暴富，无系统）
2. 总控 Agent 分发任务：
   - 世界观 Agent 生成 3 套选题 + 商业分析
   - 用户选定选题后，生成完整世界观、力量/财富体系
   - 人物 Agent 产出主角、核心配角、前 50 章反派档案
   - 大纲 Agent 输出全书总纲 + 前三卷分章框架
   - 节奏 Agent 校验大纲爽点分布，补齐缺失冲突
3. 用户确认框架，存入数据库与向量库
4. 写作 Agent 根据大纲生成第一章
5. 润色 Agent 优化文字、强化结尾钩子
6. 合规 Agent 筛查敏感内容，输出成品章节

### 流程 2：续写已有小说（最常用场景）
1. 用户上传已写章节/输入当前剧情进度
2. 总控 Agent 自动检索向量库：人设、伏笔、战力、前文剧情
3. 节奏 Agent 分析上一章结尾，设计本章冲突与收尾钩子
4. 写作 Agent 生成下一章正文
5. 自动校验人设 OOC、战力崩坏、伏笔矛盾
6. 润色优化，合规检测后返回用户

### 流程 3：卡文修复场景
1. 用户描述卡文节点：当前剧情、卡住矛盾、想要风格
2. 节奏 Agent 输出 3 套冲突解决方案，标注爽度优劣
3. 用户选定方案后，自动改写对应段落，衔接前后剧情

## 关键技术难点与解决方案

| 难点 | 解决方案 |
|------|----------|
| 长篇百万字设定遗忘、前后矛盾 | 强制双存储校验机制，所有生成前置检索向量库 + 结构化人设表，出现冲突直接弹窗提示 |
| AI 写网文平淡，缺少网文专属爽感与钩子 | 内置独立爽点节奏子 Agent，沉淀上万条网文成熟桥段模板，建立标准化章节情绪曲线约束 Prompt |
| 人物对话 OOC，性格前后不一致 | 人物档案结构化存储，每次生成对话前拉取完整角色性格、口头禅、软肋，使用对比校验器 |
| 生成内容同质化、像 AI 文，缺少个人风格 | 向量库记忆用户过往修改记录，学习用户增删偏好；支持上传作者自己写的原稿，做风格蒸馏 |
| 内容违规，平台锁文 | 独立合规 Agent 分层校验，第一层内置 Prompt 过滤毒点，第二层调用外部违禁词工具二次筛查 |

## ADDED Requirements

### Requirement: 多智能体协作架构
系统 SHALL 实现 1 总控 + 6 子 Agent 的多智能体协作架构：
- 总控调度 Agent：任务分发、上下文统筹、冲突校验
- 6 个垂直专业子 Agent：世界观&大纲规划、人物塑造、剧情爽点节奏、正文写作分镜、润色&文风校准、合规&网文运营

#### Scenario: 用户请求续写章节
- **WHEN** 用户请求续写下一章
- **THEN** 总控 Agent 依次调度：节奏 Agent 设计冲突钩子 → 写作 Agent 生成正文 → 润色 Agent 优化文字 → 合规 Agent 筛查敏感内容

### Requirement: 双存储层（结构化 DB + 向量知识库）
系统 SHALL 实现双存储层：
- 结构化数据库：存储小说基础信息、角色档案、大纲、时间线、伏笔、章节原文
- 向量知识库：存储全文段落向量、赛道范文、爽点案例、用户修改偏好

#### Scenario: Agent 生成内容前检索
- **WHEN** 任意子 Agent 准备生成内容
- **THEN** 总控 Agent 先执行向量检索（人设、前文剧情、伏笔、范文），校验设定冲突后再分发任务

### Requirement: 网文专属能力
系统 SHALL 内置网文专属能力：
- 黄金三章规则（前三章必须有钩子、冲突、金手指、期待感）
- 章节结构模板（每章结尾留钩子，每 3-5 章一个小爽点，20 章一个小高潮）
- 爽点分类库（打脸、逆袭、暴富、修为暴涨等）
- 毒点规避库（圣母主角、降智女主、憋屈无反击等）
- 情绪曲线控制（压抑铺垫→爆发爽感→短暂留白→新冲突引入）

#### Scenario: 章节爽点评分
- **WHEN** 用户提交章节请求评分
- **THEN** 剧情爽点节奏 Agent 输出爽度、钩子强度、节奏缺陷评分及优化建议

### Requirement: 外部工具插件扩展
系统 SHALL 支持外部工具插件扩展：
- 违禁词检测 API
- 网文榜单爬虫插件
- 字数统计工具
- 思维导图导出插件
- 角色关系图生成
- 翻译工具（古言白话文互转）
- 查重工具

#### Scenario: 合规检测
- **WHEN** 章节生成完成
- **THEN** 合规 Agent 调用违禁词检测 API 进行实时文本风控，输出风险报告及修改方案

### Requirement: 三种核心工作流
系统 SHALL 支持三种核心工作流：
- 新建小说（从零创作）：选题→世界观→人设→大纲→正文→润色→合规
- 续写章节（最常用）：检索向量库→设计冲突钩子→生成正文→校验→润色→合规
- 卡文修复：描述卡文节点→输出冲突解决方案→改写段落→衔接前后剧情

#### Scenario: 卡文修复
- **WHEN** 用户描述卡文节点
- **THEN** 节奏 Agent 输出 3 套冲突解决方案，用户选定后自动改写对应段落

## Impact
- Affected specs: 无（全新项目）
- Affected code: 前端项目结构、后端项目结构、数据库设计、向量知识库集成、LLM 集成层、外部工具插件
