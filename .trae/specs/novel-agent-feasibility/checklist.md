# 验收检查清单

## 可行性验证

- [ ] 后端项目能正常启动，提供 REST API
- [ ] 前端项目能正常启动，与后端联调通畅
- [ ] 小说要素配置能正确保存并生成结构化 Markdown 上下文
- [ ] 章节规划系统能根据大纲自动生成章节 TodoList
- [ ] 单角色创作引擎能基于上下文生成符合设定的小说内容
- [ ] 多角色交互引擎能在同一时间线上生成多角色行为
- [ ] 用户干预机制能暂停/继续创作并插入指令
- [ ] 角色一致性在 5000 字以上的创作中保持基本稳定
- [ ] Token 消耗在可控范围内，有预算预警机制

## 长篇小说上下文压缩机制验证

- [ ] 五层上下文金字塔架构正常工作（L0 全局/L1 卷级/L2 章节/L3 实体/L4 场景）
- [ ] Token 预算分配器能动态计算各层预算，总上下文控制在 18K Token 以内
- [ ] 章节摘要系统能自动生成结构化摘要（300-500 字/章）
- [ ] 卷级摘要系统能将多章摘要压缩为卷级汇总（1000-2000 字/卷）
- [ ] 向量数据库集成正常，支持语义检索相关章节摘要
- [ ] 角色状态机系统能自动更新角色状态（位置/情绪/目标/关系）
- [ ] 角色上线/下线标记机制正常工作
- [ ] 按需加载策略生效，只加载当前章节涉及角色的完整卡片
- [ ] 时间线管理器能自动提取事件并检测时间冲突
- [ ] 伏笔追踪系统能自动识别伏笔并提醒回收
- [ ] 混合检索策略（语义 + 关键词 + 结构化）检索准确率达标
- [ ] 上下文压缩算法在超出 Token 限制时能自动压缩低优先级内容
- [ ] 上下文缓存机制（Redis）正常工作，减少重复计算

## 技术验证

- [ ] LLM API 集成层支持至少 2 种模型切换
- [ ] 流式输出正常工作，用户体验流畅
- [ ] 上下文分层策略有效，不超出 Token 限制
- [ ] 时间线管理器能检测基本的事件冲突
- [ ] 摘要生成异步执行，不阻塞主创作流程
- [ ] 百万字级别小说创作时，上下文检索响应时间 < 2 秒

## 质量验证

- [ ] 在 10 万字以上的创作中，角色性格保持一致性
- [ ] 在 50 章以上的创作中，时间线无逻辑冲突
- [ ] 伏笔回收率 > 80%（已埋设伏笔在预期章节范围内被回收）
- [ ] 上下文压缩后，关键信息丢失率 < 5%
