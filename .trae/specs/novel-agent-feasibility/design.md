网文小说创作 Agent 完整设计方案
一、核心定位与需求拆解
1. Agent 核心目标
面向网文作者（新手 / 兼职 / 专职），全链路辅助网文生产：选题→大纲→人设→章节正文→剧情润色→节奏把控→热点适配→合规自查，支持男频 / 女频 / 现言 / 古言 / 玄幻 / 无限流等全赛道，具备长期剧情记忆、人设一致性、网文商业逻辑三大核心能力。
2. 用户核心痛点（Agent 解决方向）
卡文、剧情逻辑崩、人设 OOC；
不会写钩子、爽点、情绪铺垫，流量差；
长篇大纲难梳理，伏笔前后遗忘；
批量更新效率低，重复描写水字数；
不懂平台规则，容易违规被锁文；
不会适配榜单热点、调整剧情走向。
3. Agent 能力分层
基础层：文字生成、改写、扩写、缩句、校对；
专业网文层：爽点设计、节奏控制、钩子制造、网文文风适配；
记忆管理层：全局世界观、人物档案、剧情时间线、伏笔库；
决策规划层：分卷大纲、章节分配、冲突迭代、剧情分支推演；
工具扩展层：合规检测、数据对标、角色对话生成、标题简介产出。
二、整体架构设计（模块化 Agent）
采用多智能体分工协作架构，一个总控调度 Agent + 6 个垂直专业子 Agent，搭配向量记忆库、工具插件、用户交互层。

模块详细说明
（一）总控调度 Agent：中枢核心
核心职能
解析用户自然语言需求，拆分细分任务分发给对应子 Agent；
统一管理全局记忆，保证所有子 Agent 共享世界观、人设、已写剧情；
冲突校验：检测新生成内容是否人设崩塌、伏笔矛盾、战力崩坏；
多轮对话状态维护：记录当前创作阶段（选题 / 写大纲 / 写第一章 / 卡文修改）；
汇总各子 Agent 输出，整合交付给用户。
内置提示词约束
强制优先读取向量库历史设定，所有生成内容必须和已有设定对齐；
区分用户指令优先级：人设＞世界观＞主线剧情＞临时剧情；
识别用户诉求类型：新建小说 / 续写章节 / 改剧情 / 卡文求助 / 优化爽点。
（二）六大垂直专业子 Agent 设计
1. 世界观 & 大纲规划 Agent
负责小说顶层框架，解决 “故事讲什么”
功能清单
赛道选题生成：根据用户偏好（玄幻 / 甜宠 / 无限 / 都市战神）输出热门选题、对标榜单爆款；
世界观搭建：力量体系、地域地图、势力划分、历史背景、规则限制；
分级大纲：全书总纲→分卷大纲→每卷核心冲突→百章级长线伏笔；
剧情分支推演：给出多条剧情走向，标注每条分支爽度、受众适配度；
节奏框架模板：网文通用黄金结构（开篇金手指→第一次冲突→升级→大反派→阶段性高潮）。
输出结构化格式示例
plaintext
【第一卷大纲：废柴觉醒】
卷核心目标：主角获得金手指，摆脱屈辱，打脸宗门仇人
卷10章核心节点：
第1章：楔子羞辱钩子（开篇冲突）
第3章：金手指激活（第一个爽点）
第7章：宗门大比当众打脸（小高潮）
伏笔预埋：失踪母亲留下的玉佩暗藏上古血脉（第三卷回收）
战力梯度：炼气3层→炼气7层
2. 人物塑造 Agent
杜绝 OOC，统一全文人设逻辑
存储结构化人物档案（存入数据库）
每个角色固定字段：
基础：姓名、年龄、外貌、身份、初始立场
内核：执念、软肋、性格底色、说话语气、行为习惯
能力：战力 / 技能、金手指、短板、成长曲线
关系网：仇敌、爱人、师徒、盟友、隐藏卧底身份
标志性台词、专属动作、情绪触发反应
核心能力
批量生成主角 / 配角 / 反派 / 工具人；
校验正文对话是否符合人设，自动修正崩人设台词；
角色成长线设计：每一卷人物心态变化；
反派逻辑合理化，避免单纯 “为坏而坏”。
3. 剧情爽点节奏 Agent（网文核心差异化模块）
网文和普通写作 AI 最大区别，专门适配平台读者阅读习惯
内置网文底层规则库（Prompt 内置）
黄金三章规则：前三章必须有钩子、冲突、金手指、期待感；
章节结构模板：每章结尾留钩子，每 3-5 章一个小爽点，20 章一个小高潮；
爽点分类库：打脸、逆袭、暴富、修为暴涨、美人倾心、反派后悔、秘境寻宝、预知未来等；
毒点规避库：圣母主角、降智女主、憋屈无反击、战力崩坏、烂俗误会；
情绪曲线控制：压抑铺垫→爆发爽感→短暂留白→新冲突引入。
功能
针对现有章节打分：爽度、钩子强度、节奏缺陷；
卡文救场：给出 3 种化解当前僵局的冲突方案；
高潮段落强化：放大情绪，增加读者代入感；
调整剧情憋屈程度，平衡压抑与爽感。
4. 正文写作分镜 Agent
落地产出章节文字，分镜头式写作，适配网文分段习惯
功能
根据大纲、人设、节奏要求，生成完整章节正文；
网文分段逻辑：短句、少大段文字，对话单独分行，适配手机阅读；
自动穿插细节：环境描写、心理活动、动作神态，不空洞流水账；
区分视角：第三人称全能视角 / 主角第一视角；
批量续写：支持一次性生成 3/5/10 章内容，保持剧情连贯。
内置约束
禁止过度文绉绉书面化语言，严格匹配赛道文风（玄幻热血、现言甜宠、悬疑惊悚）。
5. 润色 & 文风校准 Agent
文字优化、统一文风、解决水文字 / 干巴巴问题
功能
多档位改写：精简缩句、扩写填充细节、强化情绪、淡化狗血；
文风统一：前后章节用词、对话风格对齐；
消除重复描写：重复动作、重复心理、重复场景；
标题 / 简介 / 章节名生成：适配平台引流，制造悬念；
段落打磨：优化结尾钩子，提升追读欲望。
6. 合规 & 网文运营 Agent
风控 + 商业化辅助，避免作者踩坑
合规检测能力
违禁词筛查：暴力、色情、涉政、敏感历史、不当价值观；
网文红线校验：禁止过度血腥、校园霸凌美化、乱伦、抹黑公职等；
自动替换敏感表述，给出修改方案。
运营辅助能力
对标同赛道爆款，给出剧情优化建议；
读者评论模拟推演：预判读者吐槽点、喜爱点；
推荐引流文案、书评、新书简介优化。
三、持久存储层（保障长篇不崩设定）
分为结构化数据库 + 向量知识库，是长篇网文 Agent 的核心壁垒。
1. 结构化关系数据库（SQL/JSON 存储）
固定数据表：
novel_info：小说基础信息（赛道、书名、总卷数、核心金手指）
character_table：全角色档案（上文完整人设字段）
volume_outline：分卷、分章大纲、节点冲突
plot_timeline：时间线、重大事件、战力等级体系
foreshadow_lib：伏笔清单（预埋章节、回收章节、对应线索）
chapter_content：已生成章节原文、修改记录
2. 向量知识库（向量数据库：Chroma/FAISS/Milvus）
存储非结构化文本，支持语义检索：
全文段落向量：续写时检索前文相似场景、人物互动，保持文风统一；
赛道范文向量库：海量高分网文片段，作为风格参考；
爽点 / 钩子案例库：语义匹配自动复用成熟剧情模板；
用户历史修改偏好：记住作者喜欢的叙事风格、爽点类型。
读取逻辑
每次任意子 Agent 生成内容前，总控 Agent 先执行向量检索：
检索当前出场人物完整人设；
检索前文相关剧情、已埋伏笔；
检索同赛道同类场景优质范文参考；
校验是否存在设定冲突，冲突则阻断生成，提示用户修正设定。
四、外部工具插件扩展集
可按需接入，增强 Agent 实用能力：
违禁词检测 API：实时文本风控；
网文榜单爬虫插件：自动抓取起点 / 番茄 / 七猫爆款标签、热门题材；
字数统计工具：分章节统计、每日更新字数规划；
思维导图导出插件：把大纲转为可视化思维导图；
角色关系图生成：输出人物关系网络；
翻译工具：古言白话文互转、古风专有名词释义；
查重工具：检测生成内容重复度，避免平台判定抄袭。
五、完整创作工作流（用户实操流程）
流程 1：新建一本小说（从零创作）
用户输入需求：赛道 + 偏好（例：男频都市，重生商界，低调暴富，无系统）；
总控 Agent 分发任务：
世界观 Agent 生成 3 套选题 + 商业分析；
用户选定选题后，生成完整世界观、力量 / 财富体系；
人物 Agent 产出主角、核心配角、前 50 章反派档案；
大纲 Agent 输出全书总纲 + 前三卷分章框架；
节奏 Agent 校验大纲爽点分布，补齐缺失冲突；
用户确认框架，存入数据库与向量库；
写作 Agent 根据大纲生成第一章；
润色 Agent 优化文字、强化结尾钩子；
合规 Agent 筛查敏感内容，输出成品章节。
流程 2：续写已有小说（最常用场景）
用户上传已写章节 / 输入当前剧情进度；
总控 Agent 自动检索向量库：人设、伏笔、战力、前文剧情；
节奏 Agent 分析上一章结尾，设计本章冲突与收尾钩子；
写作 Agent 生成下一章正文；
自动校验人设 OOC、战力崩坏、伏笔矛盾；
润色优化，合规检测后返回用户。
流程 3：卡文修复场景
用户描述卡文节点：当前剧情、卡住矛盾、想要风格；
节奏 Agent 输出 3 套冲突解决方案，标注爽度优劣；
用户选定方案后，自动改写对应段落，衔接前后剧情。
六、关键技术难点与解决方案
难点 1：长篇百万字设定遗忘、前后矛盾
方案：强制双存储校验机制，所有生成前置检索向量库 + 结构化人设表，出现冲突直接弹窗提示，不生成矛盾内容；新增 “设定修改模块”，修改人设 / 世界观后自动同步更新全库。
难点 2：AI 写网文平淡，缺少网文专属爽感与钩子
方案：内置独立爽点节奏子 Agent，沉淀上万条网文成熟桥段模板，建立标准化章节情绪曲线约束 Prompt，强制每章结尾设置悬念钩子。
难点 3：人物对话 OOC，性格前后不一致
方案：人物档案结构化存储，每次生成对话前拉取完整角色性格、口头禅、软肋，使用对比校验器，识别不符合人设内容并自动重写。
难点 4：生成内容同质化、像 AI 文，缺少个人风格
方案：向量库记忆用户过往修改记录，学习用户增删偏好；支持上传作者自己写的原稿，做风格蒸馏，让 AI 贴合作者文笔。
难点 5：内容违规，平台锁文
方案：独立合规 Agent 分层校验，第一层内置 Prompt 过滤毒点，第二层调用外部违禁词工具二次筛查，敏感内容自动给出替换方案。
七、Prompt 分层设计核心示例（总控 Agent 基础提示词）
plaintext
你是网文创作总控Agent，统筹多个专业写作子智能体，严格遵循以下规则：
1. 所有生成内容必须优先读取向量知识库内存储的小说设定：世界观、人物档案、分卷大纲、已埋伏笔、战力体系；
2. 识别用户当前创作阶段：选题阶段/搭建大纲/塑造人物/撰写章节/卡文修改/润色优化；
3. 分发任务给对应垂直子Agent，整合输出内容前执行三重校验：
   ①人设一致性校验，禁止角色行为、语言违背档案设定；
   ②剧情逻辑校验，不能出现战力崩坏、伏笔冲突、时间线混乱；
   ③网文节奏校验，符合黄金三章、章节钩子、爽点分布规则；
4. 区分赛道文风：玄幻热血、古言权谋、现言甜宠、无限流惊悚、都市重生等，不混用语言风格；
5. 规避网文毒点：无无端憋屈、无强行降智、无圣母无底线原谅反派；
6. 生成完成后交由合规Agent筛查敏感、违禁内容，有风险则提供修改方案。
