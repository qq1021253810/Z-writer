# 双模式小说信息搜集 Spec

## Why
当前系统仅支持表单填写方式搜集小说信息，用户需要一次性理解所有字段含义并填写。对于新手用户或不明确自己需求的用户来说，门槛较高。新增对话问答引导模式，通过细化的对话步骤逐步引导用户完成信息搜集，并在每个步骤提供"自动生成"选项，降低使用门槛，提升体验。

## What Changes
- 在创建小说页面新增"模式选择"功能：表单模式 / 对话模式
- 新增对话引导式信息搜集前端页面和状态管理
- 后端保持现有 `CreateNovelWorkflow` 不变，两种模式最终提交相同的 `CreateNovelRequest`
- 新增对话引导策略服务，根据用户回答智能提取和补全小说信息
- 对话模式下支持实时预览已收集的信息
- 每个引导步骤支持"自动生成"选项，AI 生成内容后自动进入下一步

## Impact
- Affected specs: novel-agent-feasibility（扩展 Requirement: 三种核心工作流）
- Affected code: 
  - 前端：`CreateNovel.tsx`（改造为支持双模式）、新增对话引导组件
  - 后端：新增对话引导服务类，复用现有 Agent 能力
  - 路由：复用 `/create` 路由，内部通过 Tab 切换

## ADDED Requirements

### Requirement: 双模式切换
系统 SHALL 在小说创建页面提供两种信息搜集模式：
- **表单模式**：现有的表单填写方式，适合有经验的用户
- **对话模式**：通过对话问答引导用户逐步填写，适合新手或不确定需求的用户

#### Scenario: 用户选择对话模式
- **WHEN** 用户在创建页面点击"对话引导"模式
- **THEN** 页面切换到对话界面，显示欢迎语和第一个引导问题

### Requirement: 对话引导流程（细粒度步骤）
系统 SHALL 在对话模式下按照以下顺序引导用户，共 10 个步骤：

| 步骤 | 引导内容 | 自动生成支持 |
|------|----------|-------------|
| 1 | 欢迎 + 了解写作意图 | 无 |
| 2 | 小说赛道选择（提供选项卡片） | 基于意图推荐赛道 |
| 3 | 核心想法/故事梗概 | 基于意图扩写梗概 |
| 4 | 小说标题（生成 3-5 个候选标题供选择） | 基于梗概生成标题 |
| 5 | 小说简介（生成 3 个版本供选择） | 基于梗概+赛道生成简介 |
| 6 | 世界观 - 力量/能力体系 | 基于赛道生成力量体系 |
| 7 | 世界观 - 世界背景/地图 | 基于赛道生成世界背景 |
| 8 | 金手指/主角特殊设定 | 基于已有信息生成金手指 |
| 9 | 预计卷数（提供建议值） | 基于故事复杂度建议卷数 |
| 10 | 信息确认（展示汇总，可返回修改） | 无 |

#### Scenario: 用户选择自动生成
- **WHEN** 用户在当前步骤点击"帮我生成"按钮
- **THEN** 系统调用 AI 生成内容，显示生成结果，用户可以选择"采用并下一步"或"重新生成"

#### Scenario: 用户手动输入
- **WHEN** 用户在当前步骤手动输入/选择内容
- **THEN** 系统保存内容，用户可以点击"下一步"或"跳过"

#### Scenario: 用户跳过步骤
- **WHEN** 用户点击"跳过"按钮
- **THEN** 该步骤标记为"待补充"，系统进入下一个步骤

#### Scenario: 对话引导完成信息收集
- **WHEN** 用户完成第 10 步（信息确认）并点击"开始创作"
- **THEN** 系统提交创建请求，调用 `CreateNovelWorkflow`

### Requirement: 对话实时预览
系统 SHALL 在对话界面侧边（桌面端）或底部（移动端）实时展示已收集的信息：
- 已完成的步骤显示绿色标记和具体内容摘要
- 已跳过的步骤显示橙色标记"待补充"
- 当前步骤显示蓝色标记"进行中"
- 未到达的步骤显示灰色标记"未开始"

#### Scenario: 用户查看已收集信息
- **WHEN** 用户在对话过程中
- **THEN** 预览区域实时更新，点击已完成步骤可返回修改

### Requirement: 对话灵活性
系统 SHALL 允许用户在对话过程中：
- 跳过某个问题稍后补充
- 点击预览面板中的已完成步骤，返回修改
- 随时切换到表单模式，且已收集的数据自动填充到表单
- 在表单模式下修改后，可切换回对话模式继续

#### Scenario: 用户修改已回答问题
- **WHEN** 用户点击预览面板中已完成的步骤
- **THEN** 系统回到对应的引导问题，允许用户重新回答，更新后自动重新计算后续步骤

### Requirement: 自动生成 API
系统 SHALL 提供对话引导生成 API，支持按步骤生成内容：

| API | 方法 | 说明 |
|-----|------|------|
| `/api/dialogue-guide/generate` | POST | 根据已有信息和步骤类型生成内容 |

请求参数：
```json
{
  "step": "title",
  "existingInfo": {
    "genre": "玄幻",
    "synopsis": "...",
    ...
  }
}
```

响应：
```json
{
  "success": true,
  "suggestions": ["标题1", "标题2", "标题3"],
  "generatedContent": "..."
}
```

## MODIFIED Requirements

### Requirement: 三种核心工作流
**原描述**：
系统 SHALL 支持三种核心工作流：新建小说（从零创作）、续写章节、卡文修复

**修改后**：
系统 SHALL 支持三种核心工作流，其中"新建小说"工作流支持两种信息搜集方式：
- 表单模式（直接填写）
- 对话模式（引导问答，支持自动生成）
两种方式最终都调用相同的 `CreateNovelWorkflow`，提交相同的请求参数。

## Impact
- 前端需要新增对话引导 UI 组件和状态管理，包含 10 个引导步骤
- 后端需要新增对话引导 API，复用现有 Agent 能力生成各步骤内容
- 两种模式共享相同的 API 接口 `POST /api/workflow/create-novel`
